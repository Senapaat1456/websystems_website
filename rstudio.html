R Studio
R Studio is an IDE (Integrated Developement Enviorment) for writing code in R. R is a programing language similar to Python, but with a larger emphasis on statistics and data science. The DAA uses R and R Studio for most of their projects and competitions. R Studio has a lot of well defined features that the guide won't go over, but almost everything you could want to do with an IDE has an inplementation in R Studio. This includes functionalities such as Version Control (Git), importing and exporting data sets, and distrubted computing. A great resource is rdocumentation.org. Another great resource is stackoverflow.com, if you have a question, most likly some else already has had a similar question and posted it to stackoverflow.
This guide will teach you how to:
  1. Install R Studio
  2. Create a new R Notebook
  3. Install a new Package
1. Installing R Studio
R Studio is simple to install. The steps are:
  1. Navegate to rstudio.com. This is the primary host for R Studio.
  2. Click 'Download' in the top right corner.
  3. Click to download the Open Source version of R Studio.
  4. Select and download the version of R Studio for your computer under the header 'All Installers'
  5. Lastly, run in the installer.
  6. If it works, when you launch R Studio, you should be greated by the homepage.
2. Creating a new R Notebook
Notebooks in R are helpful when you have lots of code for a given project that you only want to run parts of at a time. You can make 'chunks' of code that can be run independent of each other. Since R has minimal pre-runtime syntax checks, having your code broken into chunks helps avoid your code failing entirly when a single line of code has an error. To create a new notebook:
  1. Click on 'file' in the top-left corner
  2. Click on 'new file'
  3. Click on 'R Notebook'

Like everything in R Studio there is multiple ways of doing things. There are at least 3 different ways to make a new Notebook, and this is just 1 of them. Never be afraid to expirment and use which ever method you like best. R Studio is a tool designed to be versitile and customizeable. You also might notice some exaple text in your Notebook when you first create it. Feel free to delete it all. You can disable this in the R Studio Settings.

To create a chunk the format is as follows. Notice that the charcters that comes before the brackets are the ` characters, also known as the grave accent not a single quote. The key for that above the tab key on most keyboards. The 'r' in the bracket says the programing language in the chunk is r (you can use python and other simlar langages in an R notebook too) and the everything that follows the r in the backets is the chunk name, this can be as short or as long as you want. You close a chunk with 3 more grave accents

3. Instaling Packages in R Studio
The last thing I am going to show you is how to install a package in R Studio. Like many programing langages, the base language of R only has so many features, and you will hit a point where you want to do something that is not suported by base R. When you hit this point you have two options, one is to program that functionality your self, or know that someone else has already done it and published it.
  1. To install the package, first you need to know what package has what you need. Ussualy a quick search on the internet can help you figure this out. For this example we will be adding the package 'XGBoost'
  2. Start by going over the the right menu of R Studio and make sure the tab 'Packages' is open.
  3. Then click 'Install Packages'
  4. Type the name of package into the textbox and make sure that 'Install Dependencies' is clicked.
  5. Click Install. Then wait.
  6. Once it is done downloading, you need to activate it. There is two ways to do this. The first is under the packages tab, find the package in the list and just check the box next to the name. This presents with two issues, packages don't automatically load when you start R Studio so you would have to do this every time, and if you send your code to another programer, they won't know what packages you use. That is why you should use the 'library()' function. At the start of each chunk of code add the line 'library(package1,package2,...)' and list the packages you need. This will load the packages and let others know they need to install them if they are missing.
