Expectations and how I met them, in order:

  1. The Project is for the Data Analytics Assocation, an on-campus group.
  2. Each page of the site has a header.
  3. The site has five separate pages including: Homepage, R Guide, GGPlot2 Guide, Calander, and Leadership Page
  4. There are links through the pages including in the header.
  5. The site has a separate layout for small and large screens.
  6. There is a master style sheet to keep a consistent style across the pages
  7. The cite has images on almost all its pages, and each image has its own alternate text for accessibility. The scrolling slide show on the home page does automatically update the alternate text when the images change.
  8. Each page has a custom favicon (it’s not beautiful, I am no graphic designer).
  9. Everything is sorted into proper folders, including subfolders for the images.
  10. The site is not built on any libraries or frameworks.
  11. I tested all html and css files and they all passed without any issues.
  12. I shared the git repository with you.
  13. All file names are lowercase and alphanumeric.
  14. The website is served out of port 8443.
  15. It is a https connection.
  16. The content is neatly aligned.
  17. The site has two different fonts, sans-serif for the headings and serif for the body.
  18. The “GGPlot2 Basics” page has the user submit information to generate R code.
  19. The “Installing R” page has many links to outside resources for R.
  20. The code is readable.
  21. The homepage as a rotating slide show animation made with javascript.

Feedback

From Collin Goldsworthy

  1. I like the color scheme a lot! (Keeping it minty fresh like our MUDAC presentation)
  2. I also like that you included helpful visuals, and I really like the code generator!
  3. I’d put some padding at the bottom of the home page and tutorial pages so that you can scroll up and not have text stuck at the bottom of the browser window.
  4. I like the idea of the random pictures on the home page, but the way they change feels a little awkward. A cool middle ground might be to just have it randomly select 3 images each time the user goes to the homepage.
  5. Maybe make the logo on the top be a link to the homepage since that’s common practice. There’s a few spelling errors on each page, just copy paste your text into word and it’ll catch all of them.
  6. For the leadership page Tyler is the Secretary, Eric is the Treasurer. If you want, you could also include a picture of Abra, but that’s not 100 percent necessary.
